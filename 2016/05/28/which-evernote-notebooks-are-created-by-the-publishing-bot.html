Which Evernote Notebooks are created by the Publishing Bot ?

Posted on 28.May 2016 | Tags pubbot.faq

The Cloudburo Publishing Bot requires for the communication with Evernote a set of Evernote Notebooks per configured site. Additionally the Publishing Bot will create a set of tags which allows you to control the site publishing process.

These notebooks - as well as the tags - will be created during the initial site configuration steps. Additionally the publishing bot will check before each process run, the availability of the Notebooks and Tags in your Evernote Account and would kick off a recreation, in case you deleted or removed notebooks or tags accidentally.

Notebooks created by the Publishing Bot


The Publishing Bot will create for each site - in this example you would have chosen the name blogtest during site configuration - 5 new Evernote Notebooks. They start with the the prefix blog-

  • A-Fetch: Is a notebook required by the News Publishing Workflow. Any article place here will be full text extracted and moved to the B-Prepare notebook by the Publishing Bot. For details refer to the following How To.
  • B-Prepare: Is used by News Publishing Workflow, as well as can be seen as the notebook where you store your draft articles you are working on (this is optional, obviously you can work on a note in any kind of notebook). The Publishing bot isn’t doing anything with this notebook.
  • D-NewsGen: Move your ready-to-publish notes which are following the News Workflow to this Notebook. For details refer to the following How To. The Publishing Bot will fetch periodically the articles and publishes them, you will be informed by email when the publishing completed.
  • E-CurateGen: Move your ready-to-publish notes which are self-created or curation to this Notebook. For details refer to the following How To. The Publishing Bot will fetch periodically the articles and publishes them, you will be informed by email when the publishing completed.
  • Z-Archive: All published notes will be moved to this note book. In case you want to update some notes. Go here, update the content and move it back to the D-NewsGen or E-CurateGen notebook, dependent on the article style.
Tags created by the Publishing Bot


The Publishing Bot will create a set of Tags required for its working, they all start with the prefix clb-

  • clb-footer<#>, clb-menu<#> allows you to control the Menu- and Footer area of your site. Refer to the following How To.
  • clb-index is used to identify a page, which used as your customised index page. Refer to the following How To.
  • clb-image-* allows you to control how pictures are rendered in your site. Refer to the following How To.
  • clb-processed will be allocated to each Evernote note, which got published on a site.

Tags created by You

You can create any tags starting with your site name <sitename>-. When associated with a notes, they will be converted in a blog category, refer to the following How To.
